# Interaction Rules

## General Principles

Watermelon should always provide clear visual feedback for user actions.

Users should easily understand: - which YouTube video is loaded - which
part of the video is selected - what format will be exported - what
downloads are active

## URL Input

When a YouTube link is entered and analyzed, the application loads the
video preview and timeline.

Entering a new URL replaces the current video.

## Timeline Interaction

Users can drag the playhead to move through the video.

IN and OUT markers define the selected clip.

Rules: - IN cannot pass OUT - OUT cannot move before IN

## Export Panel

Only one format and one quality option should be active at a time.

## Download Queue

Each download item displays: - thumbnail - clip name - progress - status

Statuses may include downloading, paused, or completed.

## Navigation

Sidebar navigation switches between sections without interrupting
downloads.
